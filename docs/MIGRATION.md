# Migration Notes — June 2026

This document records the dependency upgrade and Android modernization performed
on the Wallet / Gapjyk app. It covers **what changed, why, and what to watch out
for** so the work can be reviewed and reproduced.

- **Date:** 2026-06-02
- **App version at time of migration:** `2.0.3+5`
- **Toolchain:** Flutter 3.41.9 · Dart 3.11.5 · JDK 17

---

## 1. Summary

The project no longer compiled on Flutter 3.41 and several dependencies were
multiple major versions behind. This migration:

1. Fixed the localization compile blocker (removed `flutter_gen` synthetic package).
2. Upgraded dependencies — *latest except the riskiest two* (`calendar_view`,
   `syncfusion_flutter_charts` were kept on their current major).
3. Rewrote code affected by breaking API changes.
4. Cleaned up all analyzer warnings (`flutter analyze` is now clean).
5. Modernized the Android build (Gradle / AGP / Kotlin / SDK levels).

**Result:** `flutter analyze` → *No issues found*, and
`flutter build apk --debug --flavor development` succeeds.

---

## 2. Localization compile blocker (l10n)

Flutter removed the `package:flutter_gen/gen_l10n/...` **synthetic package**.
Generated localizations now live directly in the ARB directory
(`lib/l10n/arb/`).

| File | Change |
|------|--------|
| `lib/l10n/l10n.dart` | Import/export repointed to `package:wallet/l10n/arb/app_localizations.dart` |
| `lib/l10n/custom_localization.dart` | Same import repoint + import ordering fix |
| `lib/app/view/app.dart` | Uses the re-export from `l10n.dart` (no direct arb import) |
| `lib/l10n/tk_intl.dart` | Implemented 8 `CupertinoLocalizations` getters newly required by the SDK |

The custom Turkmen `CupertinoLocalizationTk` was missing members that recent
Flutter added to the `CupertinoLocalizations` interface. Added with Turkmen
translations:

- `backButtonLabel`, `cancelButtonLabel`
- `collapsedHint`, `expandedHint`
- `expansionTileCollapsedHint`, `expansionTileExpandedHint`
- `expansionTileCollapsedTapHint`, `expansionTileExpandedTapHint`

> `lib/l10n/arb/app_localizations*.dart` are generated by `flutter gen-l10n`
> (`generate: true` in `pubspec.yaml`) and regenerate on every build.

## 4. Breaking API changes fixed

### `flutter_local_notifications` 18 → 21
All plugin methods moved from **positional to named** parameters, and
`uiLocalNotificationDateInterpretation` was removed.
File: `lib/core/notification_service.dart`

```dart
// Before
notificationsPlugin.initialize(initializationSettings, onDidReceive...: ...);
notificationsPlugin.show(id, title, body, notificationDetails());
notificationsPlugin.cancel(id);
notificationsPlugin.zonedSchedule(
  id, title, body, scheduledDate, notificationDetails(),
  androidScheduleMode: ...,
  uiLocalNotificationDateInterpretation: ...,  // removed in v21
);

// After
notificationsPlugin.initialize(settings: initializationSettings, onDidReceive...: ...);
notificationsPlugin.show(id: id, title: title, body: body, notificationDetails: notificationDetails());
notificationsPlugin.cancel(id: id);
notificationsPlugin.zonedSchedule(
  id: id, title: title, body: body,
  scheduledDate: ..., notificationDetails: notificationDetails(),
  androidScheduleMode: AndroidScheduleMode.exactAllowWhileIdle,
);
```

### `file_picker` 8 → 11
`FilePicker.platform` was removed; methods are now **static** on `FilePicker`.
File: `lib/home/presentation/settings_page.dart`

```dart
FilePicker.platform.saveFile(...)  →  FilePicker.saveFile(...)
FilePicker.platform.pickFiles()    →  FilePicker.pickFiles()
```
(Argument signatures are unchanged.)

### `very_good_analysis` 7 → 10 & Flutter SDK deprecations
| Deprecation | Fix | Files |
|-------------|-----|-------|
| `Color.withOpacity(x)` | `withValues(alpha: x)` | `notification_screen.dart`, `category_screen.dart`, `income_expense_widget.dart` |
| `Color.red/.green/.blue` (int) | `(color.r * 255).round()` etc. | `add_category_dialog.dart` |
| `Color.value` | `.toARGB32()` | `add_category_dialog.dart` |
| `ColorScheme.background/onBackground` | `surface` / `onSurface` | `color_schemes.g.dart` |
| `ColorScheme.surfaceVariant` | `surfaceContainerHighest` | `color_schemes.g.dart`, `calendar_page.dart` |
| `calendar_view` header `backgroundColor`/`iconColor` | `HeaderStyle(decoration:, leftIconConfig:, rightIconConfig:)` | `calendar_page.dart` |
| stale TODO comment / missing EOL newline | removed / added | `tk_intl.dart`, `counter_page.dart` |

## 5. Android modernization

| Item | Before | After |
|------|--------|-------|
| Gradle wrapper | 8.0 | **8.14** |
| Android Gradle Plugin | 7.3.0 | **8.11.1** |
| Kotlin plugin | 1.9.10 | **2.2.20** |
| Java source/target | 1.8 | **17** |
| `coreLibraryDesugaring` | desugar_jdk_libs 1.2.2 | **2.1.4** |
| `compileSdk` | 34 | **`flutter.compileSdkVersion` (36)** |
| `targetSdk` | 34 | **35** |
| `minSdk` | implicit | **`flutter.minSdkVersion` (24)** |
| `androidx.window` | 1.0.0 | **1.3.0** |

Other build hygiene:

- `android/build.gradle`: deprecated `rootProject.buildDir` → `layout.buildDirectory`.
- `android/app/build.gradle`: deprecated `compileSdkVersion` method → `compileSdk`;
  removed the duplicate `compileSdkVersion 34` inside `defaultConfig`; removed the
  hardcoded `kotlin-stdlib-jdk7` dependency (provided by the Kotlin plugin).
- `android/gradle.properties`: removed deprecated `android.enableJetifier=true`,
  added `android.nonTransitiveRClass=true`, raised Gradle JVM heap.
- Removed the deprecated `package="dev.merdan.wallet"` attribute from all three
  `AndroidManifest.xml` files (the namespace lives in `build.gradle` under AGP 8).

### New permissions (required by scheduled notifications)
Added to `android/app/src/main/AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
<uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM" />
<uses-permission android:name="android.permission.USE_EXACT_ALARM" />
<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
```

These back the existing `AndroidScheduleMode.exactAllowWhileIdle` scheduling and
the boot-reschedule receiver that were already declared.

## 6. Why `compileSdk` is 36 (not 35)

`targetSdk` is **35** as intended (the Play Store compliance level).
`compileSdk` had to be **36** because `flutter_local_notifications` 21 (and the
Flutter 3.41 default) publish AAR metadata that forces the app to compile against
API 36 or later. Compiling against 36 while targeting 35 is fully supported.

## 8. Known issues & follow-ups

- **Stale template tests (8 failing).** `test/app/view/app_test.dart`,
  `test/counter/view/counter_page_test.dart`, `test/counter/cubit/counter_cubit_test.dart`
  and `test/counter/bloc/counter_bloc_test.dart` are leftovers from the original
  "Very Good Core" template. They reference a `CounterPage`/`CounterCubit` the
  customized app no longer uses and don't initialize `SharedPreferences`. These
  are **pre-existing** failures (the project didn't compile before this
  migration, so the suite was already red) — not regressions. They should be
  rewritten to match the real architecture or removed.
- **Release build not verified.** The `release` build type uses
  `minifyEnabled true` with no ProGuard/R8 keep rules and requires signing
  config (`key.properties` or `ANDROID_KEYSTORE_*` env vars). R8 may strip
  `flutter_local_notifications`' reflectively-used classes — add keep rules and
  test a signed release build before publishing.
- **`calendar_view` and `syncfusion_flutter_charts`** were intentionally left on
  their current major. Upgrading them (2.x / 33.x) is a separate, higher-risk task.
